### 业务场景
    我18年三月份入职，当时入职时公司有1000多台车在青岛跑着，下面我给大家详细介绍一下业务场景
    1. 用户端
    用户端在用户注册后需要对人进行认证，驾照认证，以及押金或者使用芝麻信用担保。以上完成后（当然你只需要注册就可以看到车，用的话需要认证和押金）可通过当前定位点查看到附近的汽车，能够点击汽车下订单获得使用权，查看一些车辆状态，也可以通过扫车上的二维码得到对车辆的使用权。当然随之而来的还有订单、优惠券、支付、订单内对车辆的操控以及订单内车辆位置。后期还增加了上门送车，在线购车等功能，这些后期会根据时间推移慢慢引入。
    2. 地服端
    地服端主要作用是工作人员（之后称地服）根据自动策略或者人工调度去把车停放到指定地点，根据调度工单地服人员会得到相应的劳动报酬。这个端在工单使用过程中同样也像用户端一样可以找到车辆位置，可以操控车辆。
    3. 运营端
    顾名思义，这个就是我们运营和客服工作用的工具，主要是对用户审核，用户订单，车辆的一些处理，运营后台在用户许可的情况下是可以远程操作车辆的（我一直觉得这是有问题的设计），还有活动的一些编辑等。当然还有一些统计，是数据中心提供出来的，虽然不在此端而是在一个大屏上，但是我觉得他仍然算运营端，只是没做一起而已。
    4. 地服后台
    用于编辑自动调度逻辑，地服人员、手动调度、工单管理等。（个人觉得这是最有意思的一块业务）
